## Publishing Checklist

Before submitting to Chrome Web Store:

- [ ] Update version in manifest.json to 1.0.0 (or your launch version)
- [ ] Test extension on fresh Chrome install
- [ ] Prepare all 5 screenshots at correct dimensions
- [ ] Create promotional images (440x280 small tile minimum)
- [ ] Write/publish privacy policy if needed
- [ ] Set up support email or GitHub issues
- [ ] Prepare demo video (optional but helpful - max 1 min)
- [ ] Review manifest permissions (currently: activeTab, storage - good!)
- [ ] Test on multiple websites (especially popular ones)
- [ ] Verify keyboard shortcuts don't conflict with common sites
- [ ] Create promotional YouTube video (optional)

---

## Tips for Approval

1. **Clear value proposition**: Description should immediately explain what problem it solves
2. **Professional screenshots**: Show real-world use cases, not just feature dumps
3. **Accurate permissions**: Only request what you need (you're good here!)
4. **Privacy policy**: Even if not required, it builds trust
5. **Support contact**: Provide easy way for users to get help
6. **Demo video**: Significantly increases conversion rates
